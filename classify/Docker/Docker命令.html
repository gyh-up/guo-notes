二、Docker 镜像相关命令 #

常见的镜像操作命令如图

shell
# 查看镜像列表
# REPOSITORY ：仓库名称。
# TAG: 用于区分同一仓库中的不同镜像。如果未指定，默认为latest。 
# IMAGE ID: 每个镜像都有长为64位的HashID,用来全网标识一个镜像。该字段只展示前面一部分，因为这一部分已经足以在本机唯一标识一个镜像了。 
# CREATED: 镜像的创建时间。 
# VIRTUAL SIZE: 镜像所占用的空间大小。
docker images
# 查看镜像ID列表，通常配合批量操作来使用
docker images -q

# 镜像查找，或者上镜像仓库获取镜像名称
docker search nginx
# 拉取镜像，镜像名称格式为`名称:版本号`，如果版本号不指定则是最新的版本
docker pull nginx 

# 删除镜像
docker rmi 镜像ID/镜像名称
# 强制删除 
docker rmi -f 镜像ID/镜像名称
# 删除所有镜像
docker rmi $(docker images -q) 

# 查看 save 用法
docker save --help
# 导出镜像到磁盘，docker save -o [保存的目标文件名称] [镜像名称]
docker save -o nginx.tar nginx:latest
# 加载文件生成镜像
docker load -i nginx.tar

三、Docker 容器相关命令 #

shell
# 创建运行容器 docker run [OPTIONS] IMAGE [COMMAND] [ARG...] 
docker run --name my_nginx -p 80:80 -d nginx
# 展示全部容器列表
docker ps -a
# 展示正在运行的容器列表
docker ps
# 进入容器，退出容器不会关闭
docker exec -it my_nginx /bin/bash
# 启动容器
docker start my_nginx 
# 删除容器
docker rm my_nginx 
# 停止所有容器 
docker stop $(docker ps -a -q) 
# 删除所有容器 
docker rm $(docker ps -a -q) 
# 基于dockerfile 构建容器（主要后面有个英文句号” . ”）
# -t ，--tag list 构建后的镜像名称 
# -f， --file string 指定dockerfile文件位置。默认运行目录下的dockerfile文件
docker build . 
docker build -t my_nginx . 
docker build -t my_nginx:v2 -f /path/Dockerfile /path
# 查看容器详细信息
docker inspect my_nginx
# 查看运行容器日志，添加 -f 参数可以持续查看日志
docker logs my_nginx

run 命令常用 OPTIONS 说明：

  • **-p：**将宿主机端口与容器端口映射，冒号左侧是宿主机端口，右侧是容器端口
  • **-d：**后台运行容器，并返回容器ID；
  • **-i：**以交互模式运行容器，通常与 -t 同时使用；
  • **-t：**为容器重新分配一个伪输入终端，通常与 -i 同时使用；
  • **--name=my_nginx：**为容器指定一个名称；
  • -v： 目录映射;
  • **-e MYSQL_ROOT_PASSWORD=123456：**设置环境变量；

其他 OPTIONS 说明：

  • -a stdin：指定标准输入输出内容类型，可选 STDIN/STDOUT/STDERR 三项；
  • --dns 8.8.8.8：指定容器使用的DNS服务器，默认和宿主一致；
  • --dns-search example.com：指定容器DNS搜索域名，默认和宿主一致；
  • -h "mars"：指定容器的 hostname；
  • --env-file=[]：从指定文件读入环境变量；
  • --cpuset="0-2" or --cpuset="0,1,2"：绑定容器到指定CPU运行；
  • -m：设置容器使用内存最大值；
  • --net="bridge"：指定容器的网络连接类型，支持 bridge/host/none/container: 四种类型；
  • --link=[]：添加链接到另一个容器；
  • --expose=[]：开放一个端口或一组端口；

3、对比 #

相对于文件大小来说，save 方式导出的文件比 export 方式导出的文件大。因为 save 方式保存了镜像的历史和层(layer)，使其可以层回滚，即回滚到之前的历史层，所以 save 方式导出的文件稍微大一些。

反观 export 方式，在导出过程中丢失所有的历史，导致其不可以层回滚，导出的文件会小一些。

可以通过 docker history 镜像名 看到层关系。

4、commit #

使用镜像创建并运行一个容器，实际上是在父镜像的基础上创建一个可读写的文件层级。我们在容器里所做的修改（包括安装新的应用程序、更改系统配置），都发生在这个层级上面。

使用commit命令将容器里的所有修改提交到本地库中，形成一个全新的镜像。

shell
docker commit -m="test commit" nginx nginx:v2 
docker history nginx:v2

五、数据卷（容器数据管理） #

将容器与数据分离，解耦合，方便操作容器内数据，保证数据安全

  • 数据卷是宿主机中的一个目录或者文件
  • 当容器目录和数据卷目录绑定后，对方修改会立即同步（容器数据持久化）
  • 一个数据卷可以被多个容器挂载（容器数据共享）
  • 一个容器可以挂载多个数据卷

1、数据卷操作 #

shell
# 创建数据卷 html
docker volume create html
# 查看所有数据卷
docker volume ls
# 查看数据卷详细信息，包括关联的宿主机目录位置
docker volume inspect
# 删除指定数据卷
docker volume rm
# 除所有未使用的数据卷
docker volume prune

2、挂载数据卷 #

我们在创建容器时，可以通过 -v 参数来挂载一个数据卷到某个容器内目录，命令格式如下：

shell
# docker run -v 宿主机目录/文件：容器内目录/文件
docker run -id --name=nginx -v /docker/html:/root/html -v /docker/log:/root/log nginx
1
2

把 /docker/html 数据卷挂载到容器内的 /root/html 这个目录中，宿主机目录必须是绝对路径，如果目录不存在会自动创建。

3、数据卷容器 #

多容器进行数据交换

  • 多个容器挂载同一个数据卷
  • 数据卷容器

shell
# 创建启动c3数据卷容器，使用 –v 参数 设置数据卷
docker run –id --name=c3 –v /volume centos:7
# 创建启动 c1 c2 容器，使用 –-volumes-from 参数 设置数据卷
docker run –id --name=c1 --volumes-from c3 centos:7 
docker run –id --name=c2 --volumes-from c3 centos:7
